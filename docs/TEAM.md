## 角色定义

### 1. 项目负责人 / 架构师

**职责**：
- 把控整体技术架构与模块边界
- 评审关键资金流程代码
- 制定接口契约、错误码、数据库规范
- 决定技术选型与第三方接入方案

**当前重点**：
- 确认第一阶段的账户、支付密码、实名体系足够支撑后续模块
- 审查转账、提现、红包的资金安全

### 2. 后端开发 A：用户与账户基座

**负责模块**：
- `auth` 用户鉴权
- `users` 用户资料、实名认证、支付密码
- `accounts` 账户余额、资金流水
- `bills` 账单

**当前任务**：
- 保证用户注册/登录/实名/支付密码链路稳定
- 账户余额与流水查询接口优化
- 配合资金流模块完成记账

### 3. 后端开发 B：C 端资金流

**负责模块**：
- `transactions` 充值
- `transfers` 转账
- `withdrawals` 提现
- `red-packets` 红包
- `qr-codes` 收款码

**当前任务**：
- 完成第一阶段所有资金接口
- 所有资金操作走 Prisma 事务
- 日限额、大额风控、支付密码校验落地
- 红包 24 小时过期退回机制

### 4. 后端开发 C：商户与开放能力

**负责模块**：
- `merchants` 商户入驻
- `cashier` 统一收银台
- `open-api` 开放 API 与签名

**当前任务**：
- 第一阶段暂不启动，等 B 完成资金流后接入
- 提前约定收银台与收款码的接口契约

### 5. 后端开发 D：运营与财务

**负责模块**：
- `admin` 管理后台
- `finance` 财务统计
- `open-api.guard` 风控相关接口

**当前任务**：
- 第一阶段暂不启动
- 提前设计 admin 审核提现、查看风控事件的流程

### 6. 前端开发

**负责模块**：
- `public/` H5 钱包页面
- 后续商户后台 Web
- 后续管理后台 Web

**当前任务**：
- 对接第一阶段接口：提现、红包、收款码、转账限额
- 统一错误提示与加载状态

### 7. 测试 / 运维

**职责**：
- 编写接口测试与资金场景测试
- 搭建 CI/CD、部署脚本
- 监控日志与告警

**当前任务**：
- 为转账、提现、红包编写核心用例
- 确认 `.env`、数据库迁移流程在测试环境可复现

## 协作规则

1. **代码审查**：所有资金相关代码必须经过架构师或另一名后端开发 review。
2. **每日同步**：B 端资金流每日汇报接口变更，A 端同步账户侧是否受影响。
3. **接口契约**：跨模块调用先写接口文档（写在代码注释或临时 markdown），再实现。
4. **提交规范**：
   - `feat:` 新功能
   - `fix:` 修复
   - `refactor:` 重构
   - `docs:` 文档
5. **阶段同步**：每完成一个阶段，同时推送到 GitHub 和 GitCode 两个仓库。
